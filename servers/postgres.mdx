A Model Context Protocol server that provides read-only access to PostgreSQL databases. This server enables LLMs to inspect database schemas, execute read-only queries, and analyze database performance.

<Note>
All queries are executed within a READ ONLY transaction to ensure database safety.
</Note>

## Installation

<CodeGroup>

```bash Docker
docker build -t mochoa/mcp-postgres -f src/postgres/Dockerfile .
```

```bash NPM
npm install @marcelo-ochoa/server-postgres
```

</CodeGroup>

## Configuration

### Environment Variables

The PostgreSQL server uses environment variables or the `pg-connect` tool for secure credential management:

- `PG_USER`: PostgreSQL username (optional if using `pg-connect`)
- `PG_PASSWORD`: PostgreSQL password (optional if using `pg-connect`)
- `PG_SSL`: Enable SSL/TLS encryption (set to `true` to enable)

### Connection String

The connection string should contain only the host, port, and database information (without embedded credentials). Providing it as a command-line argument is **optional**. If omitted at startup, you must use the `pg-connect` tool to establish a connection before using other functionality.

**Supported connection string formats:**
- `postgresql://host:port/dbname`
- `host:port/dbname`

**SSL/TLS Encryption:**
To enable encryption, append `?sslmode=require` to the connection string or set the `PG_SSL` environment variable to `true`.

## Usage

### Claude Desktop

<Steps>

<Step title="Locate your Claude Desktop config file">

- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

</Step>

<Step title="Add the PostgreSQL MCP server configuration">

<CodeGroup>

```json Docker
{
  "mcpServers": {
    "postgres": {
      "command": "docker",
      "args": [
        "run", 
        "-i", 
        "--rm",
        "-e", "PG_USER=myuser",
        "-e", "PG_PASSWORD=mypassword",
        "mochoa/mcp-postgres",
        "postgresql://host.docker.internal:5432/mydb"
      ]
    }
  }
}
```

```json NPX
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y",
        "@marcelo-ochoa/server-postgres",
        "postgresql://localhost:5432/mydb"
      ],
      "env": {
        "PG_USER": "myuser",
        "PG_PASSWORD": "mypassword"
      }
    }
  }
}
```

</CodeGroup>

<Note>
When running Docker on macOS and connecting to PostgreSQL on your host machine, use `host.docker.internal` instead of `localhost`.
</Note>

<Warning>
Replace the following placeholders with your actual values:
- `myuser` and `mypassword` with your PostgreSQL credentials
- `localhost:5432` with your PostgreSQL server host and port
- `mydb` with your database name
</Warning>

</Step>

<Step title="Restart Claude Desktop">

Restart Claude Desktop to load the new configuration.

</Step>

</Steps>

### VS Code

Add this configuration to your VS Code User Settings (JSON) or `.vscode/mcp.json`:

<Note>
The `mcp` key is not needed in the `.vscode/mcp.json` file.
</Note>

<CodeGroup>

```json Docker
{
  "mcp": {
    "inputs": [
      {
        "type": "promptString",
        "id": "pg_user",
        "description": "PostgreSQL username"
      },
      {
        "type": "promptString",
        "id": "pg_password",
        "description": "PostgreSQL password",

    "servers": {

        "args": [
          "run",
          "-i",
          "--rm",

          "-e", "PG_PASSWORD=${input:pg_password}",

        ]
      }
    }
  }
}
```

## Features

### Tools

#### pg-query

Execute read-only SQL queries against the connected database.

**Input:**
- `sql` (string): The SQL query to execute

**Example:**
```sql
SELECT * FROM users WHERE created_at > NOW() - INTERVAL '7 days'
```

<Note>
All queries are executed within a READ ONLY transaction.
</Note>

#### pg-stats

Get statistics for a specific table.

**Input:**
- `name` (string): The name of the table to get statistics for

#### pg-explain

Explain plan for a given SQL query.

**Input:**
- `sql` (string): The SQL query to explain

#### pg-connect

Connect to a PostgreSQL database.

**Inputs:**
- `connectionString` (string): The PostgreSQL connection string without credentials (e.g., `postgresql://host:port/dbname` or `host:port/dbname`)
- `user` (string): The PostgreSQL username
- `password` (string): The PostgreSQL password

**Example:**
```
pg-connect host.docker.internal:5432/postgres postgres pg_2025
```

#### pg-awr

Generate a PostgreSQL performance report similar to Oracle AWR.

**Includes:**
- Database statistics
- Top queries (requires `pg_stat_statements` extension)
- Table/index statistics
- Connection info
- Optimization recommendations

<Warning>
For optimal performance monitoring, ensure the `pg_stat_statements` extension is enabled in your PostgreSQL database.
</Warning>

### Resources

The server provides schema information for each table in the database:

**Table Schemas** (`postgres://<dbname>/<table>/schema`)
- JSON schema information for each table
- Includes column names and data types
- Automatically discovered from database metadata
